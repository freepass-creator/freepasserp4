# 디자인 일관성 규격 (freepasserp4)

> 작업 전 필독. `CURSOR.md`(업무지시서)와 함께 본다.
> **이 문서의 목적:** 화면마다 미묘하게 달라지는 것을 막는다. 숫자·색을 손으로 찍는 순간 전체 톤이 깨진다.

---

## 0. 절대 원칙

**숫자와 색을 직접 쓰지 않는다. 토큰을 쓴다.**
토큰에 없으면 → **만들지 말고 보고**한다. (새 값 도입 = 디자인 판단 = 커서 레인 아님)

---

## 1. 토큰 (이것만 쓴다)

```ts
import { C, R, NUM, FS, ctrlH, ctrlFs, ctrlInputFs, ctrlChipH } from '@/components/ui';
```

### 1-1. 색 — `C.*`
| 용도 | 토큰 |
|---|---|
| 본문 글자 | `C.ink` |
| 보조 글자 | `C.mute` / `C.sub` |
| 흐린 글자(메타) | `C.faint` |
| 테두리 | `C.line` / `C.line2`(약) |
| 페이지 배경 | `C.bg` |
| 카드·입력 서피스 | `C.taupeBg` |
| 헤더 배경 | `C.head` |
| 호버 | `C.hover` |
| 선택 행 | `C.selected` |
| 사진·빈 서피스 | `C.placeholder` |
| 상태색 | `C.ok` `C.warn` `C.danger` `C.accent` `C.brand` |
| 틴트 배경 | `C.okBg` `C.warnBg` |

🚫 **HEX·rgba 직접 사용 금지** (`#fff`, `#f8fafc`, `rgba(...)`).
🚫 상태색을 손으로 고르지 말 것 — 톤 시스템(`Badge tone` / `toneSoft` / `toneText`)을 쓴다.
`tone` 값: `gray` `blue` `green` `amber` `red`

### 1-2. 타입 스케일 — `FS.*` (콘텐츠 글자)
| 토큰 | 크기 | 용도 |
|---|---|---|
| `FS.page` | 18 | 페이지·섹션 대제목 |
| `FS.title` | 14.5 | 목록 행 제목·패널 제목 |
| `FS.body` | 13 | 본문 |
| `FS.sub` | 12 | 보조 설명·부제 |
| `FS.cap` | 11 | 캡션·메타(시간·코드·상대) |
| `FS.micro` | 10 | 최소(뱃지 내부·마이크로 라벨) |

🚫 `fontSize: 11.5` 처럼 **숫자 직접 금지.** 중간값이 필요해 보이면 그건 대개 단계 선택이 틀린 것.

### 1-3. 컨트롤 크기 (버튼·입력·칩) — 헬퍼만
```ts
ctrlH(mobile, size)       // 높이   웹 md32/sm28 · 모바일 md40/sm36
ctrlFs(mobile, size)      // 버튼·칩 글자   모바일 16 고정
ctrlInputFs(mobile, size) // 입력·검색 글자 모바일 16 고정(iOS 확대 방지)
ctrlChipH(mobile)         // 필터칩 높이
```
🚫 `height: 32`, `fontSize: 13` 처럼 컨트롤 치수 직접 금지.
✅ 원자(`Btn`/`Input`/`Select`/`SearchInput`)를 쓰면 자동 적용된다.

### 1-4. 라운드 — `R` (=4)
모서리는 `R`. 원형은 `'50%'`. 그 외 임의값 금지.

## 2. 원자 (UI는 손으로 그리지 않는다)

| 필요 | 원자 |
|---|---|
| 한 줄 입력 | `Input` |
| 여러 줄 입력 | `Textarea` |
| 검색 | `SearchInput` |
| 버튼 / 아이콘버튼 | `Btn` / `IconBtn` |
| 선택 | `Select` |
| 상태 뱃지 / 카운트 | `Badge` / `CountPill` |
| 목록 행 | `ListRow` · `FeedListRow`(문의·계약·재고·정책) |
| 패널 | `PaneHead` / `PaneBody` / `Panel` |
| 업무 페이지 골격 | `WorkPage` + `panes`(4패널) |
| 빈 화면 | `CenterNote` |
| 로딩 | `Loading` |
| 필터칩·섹션라벨 | `FilterChips` / `SectionLabel` |
| 폼 | `FormGrid` / `FormCard` |

🚫 `<input> <textarea> <select> <button>` 직접 작성 금지 (숨김 `type="file"`만 예외).
🚫 **공용 원자를 자체 컴포넌트로 교체 금지.** 필요하면 보고.

## 3. ERP 톤 도그마

- **목록이 제품의 본체.** 행·열·**스캔성** 우선 — 한 화면에 많이 보여야 한다.
- **고밀도** 기업 백오피스. 큰 카드·큰 아이콘·과한 여백 = 잘못된 방향.
- **B2C 피드 패턴 금지**(당근류 큰 썸네일 피드). 이건 ERP 화면이다.
- 상태는 **색+뱃지**로 표현하고, 장식용 아이콘을 남발하지 않는다.
- 요약·KPI 큰 칸을 먼저 만들지 않는다. 요약은 정렬·필터로 창발시킨다.
- 숫자는 `fontFamily: NUM`(고정폭)으로 자릿수를 맞춘다.

> **사고 사례:** 목록 상태 썸네일이 72px까지 커지고 행이 88px가 되어 한 화면에 보이는 건수가 급감 → 스캔 불가 → "투박함". 44px/76px로 되돌림.

---

## 4. 모바일 규격

- 입력 글자 **16px 미만 금지** (미만이면 iOS에서 포커스 시 화면 확대 = 버그)
- 터치 타깃 **40px 이상**
- `useIsMobile()` + 위 컨트롤 헬퍼로 처리. 미디어쿼리·매직넘버 남발 금지.
- `textarea`는 브라우저 기본이 고정폭 폰트 → 반드시 `fontFamily: 'inherit'` (원자 `Textarea` 사용 시 자동)

---

## 5. 현재 위반 현황 (실측 · 점진 수렴 대상)

| 항목 | 상태 |
|---|---|
| 콘텐츠 폰트 | ❌ **19종 난립**(9~40, 0.5단위 남발) → `FS` 6단계로 수렴 |
| 하드코딩 색 | ❌ `#fff` 90회 + **일회성 색 19종**(Tailwind 팔레트 손픽) → `C.*`·tone으로 수렴 |
| 라운드 | ✅ 양호 (`R` 72회, 일회성 소수) |
| 컨트롤 치수 | ✅ 헬퍼 SSOT 존재 |
| 4패널 골격 | ✅ 4페이지 전부 `WorkPage` 준수 |

**수렴 원칙:** 한 번에 전부 바꾸지 않는다. **건드리는 파일부터** 토큰으로 교체한다.
단, **새로 쓰는 코드는 처음부터 토큰만** 쓴다(= 더 이상 나빠지지 않게).

---

## 6. 작업 후 자가점검

1. 내가 추가한 코드에 `fontSize: 숫자`가 있는가? → `FS.*`로 교체
2. `#`으로 시작하는 색이 있는가? → `C.*`로 교체
3. `height:`/`width:`로 컨트롤 크기를 지정했는가? → 원자·헬퍼로 교체
4. `<input>/<button>`을 직접 만들었는가? → 원자로 교체
5. 공용 원자를 바꾸거나 지웠는가? → 되돌리고 보고
6. 모바일에서 입력 16px·터치 40px 유지되는가?
