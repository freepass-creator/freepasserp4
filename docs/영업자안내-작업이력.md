# 영업자 업무안내·QNA (`/faq`) — 작업 이력

## 2026-07-21 신규

영업자가 계약 절차·서류·수수료를 물어볼 때마다 담당자가 직접 답하던 걸 화면으로 옮겼다.
카톡방에 이미지·PDF를 뿌리는 방법도 검토했으나, **어차피 영업자가 매일 로그인하는 곳이
ERP라서** 여기 두는 게 찾기 쉽다고 판단했다.

## 구성

| 파일 | 역할 |
|---|---|
| `lib/domain/faq.ts` | **내용 SSOT.** 문구 수정은 여기서만 한다 |
| `app/faq/page.tsx` | 화면 — 배열·검색만. 내용 하드코딩 금지 |
| `components/ui/detail.tsx` | `Disclosure` 원자 |
| `components/ui/index.tsx` | `CopyBlock` 원자 |

`GUIDE`(업무 절차 안내)는 반드시 알아야 하는 것이라 항상 펼쳐 두고,
`FAQ`(QnA)는 궁금할 때만 여는 것이라 제목만 나열하고 눌러야 펼쳐진다.
**같은 내용을 양쪽에 두지 않는다** — 한쪽만 고쳐지면 어긋난다.

## 원자를 새로 만든 이유

- **`Disclosure`** — `Sec`은 페이지 섹션용이라 숨김 레지스트리·드래그 정렬·localStorage가 붙어 있다.
  질문 30개에 쓰면 전부 개별 등록돼 엉킨다. 목록 안 한 줄짜리 접이식이 없어 새로 만들었다.
- **`CopyBlock`** — 접수 양식을 눌러서 복사. 비보안 컨텍스트에서는 textarea 폴백.

규격상 페이지에서 raw 컨트롤을 짜면 안 되므로 둘 다 원자로 올렸다.

## 노출 범위

`TopBar` GROUPS 마지막 묶음, **설정 바로 위**. `roles: ['agent', 'admin']`.
관리자를 넣은 건 내용 검수·문의 대응 때문이고, 공급사(provider)에게는 보이지 않는다.

## 수수료를 화면에 적지 않는 이유

요율이 **파트너·건별로 다르다.** 한때 특정 파트너 협의가를 그대로 적었다가,
전 영업자가 보는 화면이라 그 값이 기준으로 굳어질 위험이 있어 걷어냈다.
지금은 `지급 기준(당월 출고 건) · 지급일(익월 15일) · 요율은 별도 문의`만 남겼다.
**요율 숫자를 다시 넣자는 요청이 오면 이 결정을 먼저 상기시킬 것.**

## 내용 출처

전부 대표 확인을 거친 답변이다(2026-07-21 구두 확인).
심사 속도·무심사 기준·개인회생 가능 여부·계약금 환불·보증금 분납·주행거리·외국인 F4·
운전자 범위·연락두절 시 처리·광고 허용·재고표 공유 규칙.
**추정으로 채우지 말 것** — 영업자가 고객에게 그대로 전달하는 내용이라 틀리면 계약 사고가 된다.

## 카톡 이미지와의 관계

`C:\dev\프리패스제휴사관리\안내문\`이 이 파일을 읽어 PNG를 굽는다.

```
node dump_faq.mjs      # faq.ts → out/faq.json
python make_cards.py   # → out/*.png 12장
```

`dump_faq.mjs`가 `export const GUIDE`·`FAQ` 배열 리터럴을 대괄호 짝맞춤으로 잘라내 eval한다.
**두 상수의 이름이나 `export const` 형태를 바꾸면 그쪽이 깨진다.**
